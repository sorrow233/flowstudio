修复写作分类管理与顶部分类显示不一致，恢复分类编辑/删除生效（v2.7.2）

用户原始反馈：
“这里完全不生效，就是和写作的分类完全不一致https://flowstudio.catzz.work/writing
更改也完全没有效果，是个大问题呢”

问题根因：
1. `WritingCategoryManager` 在分类归一化时使用 `{ ...cat, ...preset }`，把默认分类 `id`（如 `draft`）错误覆盖成颜色预设 `id`（如 `stone`）。
2. 分类编辑与删除都依赖 `id` 精确匹配，因此传入了错误 `id` 后，`onUpdate/onRemove` 无法命中真实分类，表现为“更改完全不生效”。
3. 同一处 `id` 覆盖还会破坏默认分类翻译映射，导致弹窗内显示 `Draft/Plot/...`，而顶部分类显示 `草稿/情节/...`，出现“分类不一致”。
4. 新增分类时把整个颜色预设对象（含 `id`）传入，导致分类主键和颜色绑定，增加后续主键冲突与不可预期行为风险。

修复内容：
1. 新增 `pickPresetStyle`，仅抽取颜色相关样式字段（`color/dotColor/textColor/buttonClass/darkButtonClass`）。
2. 将分类归一化改为保留原始分类主键与名称，仅覆盖样式字段，避免污染业务 `id`。
3. 将新增分类逻辑改为只提交 `label + 样式`，不再提交颜色预设 `id`，由数据层生成稳定分类主键。
4. 版本号从 `2.7.1` 升级至 `2.7.2`（小功能修复版本）。

验证结果：
1. 本地执行 `npm run build` 通过。
