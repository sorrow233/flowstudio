重写写作分类管理功能与视觉交互，补齐可用性与稳定性（v2.7.3）

用户原始诉求：
“继续优化我给你的文案的功能和设计，找到你之前没有弄好的，进行全面的检查，重新设计，重写优化这次你增加的功能.确保第一版交给用户的就是一个能够使用和完善的功能。”

本次全面检查发现的未完善问题：
1. 分类管理仅支持基础增删改，缺少重名校验、长度校验、颜色操作和错误反馈，易出现不可预期输入。
2. 删除分类时会逐条触发 `onUpdate + immediateSync`，同步调用过于频繁，存在不必要的同步抖动风险。
3. 分类管理弹窗逻辑与展示耦合严重，可维护性偏低，不利于后续稳定迭代。
4. 顶部分类选择器缺少当前分类文稿数量信息，状态感知弱。
5. 部分分类相关提示文案依赖 fallback，国际化覆盖不完整。

核心重构与优化：
1. 重写 `WritingCategoryManager` 交互逻辑：
   - 新增分类名称校验（必填 / 最大长度 / 重名检测）。
   - 新增键盘交互（Enter 确认、Escape 分级退出当前操作）。
   - 新增颜色能力：
     - 自定义分类支持一键轮换颜色。
     - 新增分类支持显式选择颜色。
     - 默认分类颜色锁定，避免与系统默认语义冲突。
   - 新增分类文稿数量可视化与删除迁移提示，提升操作可预期性。
2. 重构为模块化结构，拆分“上帝文件”：
   - 新增 `WritingCategoryManagerItem.jsx`（单行分类项渲染与交互）。
   - 新增 `WritingCategoryAddPanel.jsx`（新增分类输入与配色面板）。
   - 新增 `writingCategoryManagerUtils.js`（纯函数校验与配色策略）。
3. 补充写作分类公共工具：
   - 在 `writingCategoryUtils.js` 增加 `MAX_WRITING_CATEGORIES`、默认分类识别、预设样式提取、颜色预设匹配等复用能力。
4. 优化删除分类数据流：
   - `WritingBoard` 中删除分类时改为本地批量更新分类归属后再统一同步，修复多次即时同步造成的性能与一致性风险。
5. 优化顶部分类条可用性：
   - `WritingCategorySelector` 显示“分类名 + 当前文稿数”。
   - 每个分类点位增加数量语义（title/aria-label）。
6. 国际化完善：
   - 新增中英文分类管理提示文案（名称校验、颜色提示、删除迁移提示等）。
7. 版本升级：
   - `2.7.2 -> 2.7.3`（小功能重构与可用性增强版本）。

修复的具体问题（Bug Fixes）：
1. 修复分类名可输入异常值却无提示的问题（现有必填/长度/重名校验）。
2. 修复删除分类导致同步触发次数过高的问题（改为批量更新后统一同步）。
3. 修复分类状态信息弱、难以判断分类内容量的问题（新增文稿计数展示）。
4. 修复分类管理代码耦合过高导致后续容易回归的问题（完成组件与工具模块拆分）。

本地验证：
1. `npm run build` 通过（v2.7.3）。
